入门案例

  1. 读取配置文件

  2. 创建SqlSessionFactory工厂

  • 创建工厂mybatis使用了构建者模式（把对象的创建细节隐藏，让使用者直接调用方法即可拿到对象）
  1. 使用工厂生产SqlSession对象
  • 生产SqlSession使用了工厂模式（解耦，即降低类之间的依赖关系）
  1. 使用SqlSession创建接口的代理对象

  • session.getMapper(Class type);

  • 创建接口实现类使用了代理模式（不修改源码的基础上对已有方法增强）

  1. 使用代理对象执行方法

  2. 释放资源

注意事项：不要忘记在映射配置中告诉mybatis要把数据封装到哪个实体类中（设定resultType的值为全类名）

  • 在sqlmapCongfig.xml中指定mapper 的class属性，指向注解的repository全限定类名

查询操作代码大致分析

  • 读取配置文件

    解析xml ：dom4j

  • selectList方法

    1. 根据配置文件的信息创建Connection对象（注册驱动，获取连接）

    2. 获取预处理对象PreparedStatement

    connection.preparedStatement(sql)

    1. 执行查询sql语句，返回ResultSet

    ResultSet resultSet =preparedStatement.executeQuery();

    1. 遍历结构集 resultSet 用于封装

    List list =new ArrayList();

    while(resultSet.next(){

    E element =(E) Class.forName(“全限定类名”).newInstance ()（对应xml中的resultType）

    封装:把每个内容添加到element中，再把element加入到list中

    根据反射的方式来根据名称获取属性并赋值

    list.add(element);

    }

    1. 返回list

    return list；

  • 执行上述方法需要提供两个信息：

    • 连接信息（用来创建connection）
    • 映射信息：sql语句和封装结果的实体类全限定名（resultType），把这两个信息组合起来定义成一个对象（Map）
    String Mapper（映射）
    com.litao.repository.UserRepository String ClassPath
    findall string sql
